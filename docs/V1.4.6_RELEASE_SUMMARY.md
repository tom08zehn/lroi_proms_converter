## 🎯 Issues Fixed

### Issue 1: GUI "Use default" Checkbox Not Ticked ✅

**Problem:**
```bash
python main.py --gui
# GUI opens but "Use default" for Output XML file is UNCHECKED
# Should be CHECKED since no --output was provided
```

**Root Cause:**  
In main.py, `_resolve_output_path()` was called before GUI launch and always returned a value (either from --output or from template). This value was passed to prepopulate dict, making GUI think user provided CLI argument.

**Fix:**
```python
# OLD - main.py
prepopulate = {
    "output_path": output_path,  # Always has value (from template)
}

# NEW - main.py  
prepopulate = {
    "output_path": args.output,  # None if not provided
}
```

**Result:**
- ✅ No --output → checkbox CHECKED, template shown
- ✅ With --output → checkbox UNCHECKED, CLI value shown

## 📖 Documentation Highlights

### Configuration Section (README.md)

**Before (v1.4.4 and earlier):**
```toml
# Outdated structure
[PROM.OKS]
name = "Oxford Knee Score"
col_admission_id = "Admission ID"
col_patient_id = "Patient ID"

[lut]
col_gender = "Gender"
validation_mode = "strict"  # Deprecated!
```

**After (v1.4.6):**
```toml
# Current structure
[PROM.OKS]
detection_column = "1. How would you describe..."

[PROM.OKS.lookup]
required = true
join_column = "Admission ID"
add_columns = ["Gender", "Date of Birth", "Laterality"]

[PROM.OKS.UPNNUM]
column = "Patient ID"

[PROM.OKS.GENDER]
column = "__LUT__Gender"
[[PROM.OKS.GENDER.value]]
match = "^M(ale)?$"
replace = "0"
flags = "i"
```

**Key Improvements:**
- ✅ Shows actual working syntax
- ✅ Explains `__LUT__` prefix
- ✅ Documents regex flags
- ✅ Clear distinction between XLS columns and LUT columns
